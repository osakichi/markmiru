## 5. ライセンス遵守メモ

- **共通義務**: MIT / BSD / ISC / Apache-2.0 はいずれも、著作権表示とライセンス文の**同梱（サードパーティライセンス表記）**が必要。アプリ内「ライセンス情報」やインストーラに同梱する。
- **Apache-2.0（DOMPurify を Apache 側で利用する場合）**: 上流に `NOTICE` があれば併せて同梱。
- **DOMPurify の MPL-2.0**: ファイル単位の弱いコピーレフト。**改変しなければソース開示義務は実質発生しない**。改変回避、または Apache-2.0 側を選択すれば懸念なし。
- **フォント（SIL OFL 1.1）**: フォントファイルと共に **OFL ライセンス文を同梱**。フォント単体での有償販売は不可（アプリ同梱配布は可）。予約フォント名を改変版に使わない。
- **コピーレフト（GPL 等）の依存は無し**。クローズドソースのまま配布可能。
- **推移的依存（transitive）**: markdown-it（`linkify-it`/`uc.micro` 等）、mermaid（`d3` 等）にも依存があるが、いずれも寛容ライセンス。
- **リリース前に依存ツリー全体のライセンス自動監査**を実施することを推奨。
- **highlight.js の容量対策**: 必要な言語のみを同梱するビルド構成にして配布サイズを抑える。
