# Markmiru スタイル設定設計（閲覧モード）

最終更新: 2026-05-31

閲覧モードの表示スタイルを細かく調整し、かつ**クロスプラットフォームで同一表示**を実現するための設計。全体方針は [`アーキテクチャ・画面設計.md`](アーキテクチャ・画面設計.md) を参照。

## 0. 確定方針

| 項目 | 決定 |
|------|------|
| 表現方式 | **構造化「スタイルプロファイル」(JSON) ＋ CSS 変数**で適用 |
| フォント | **同梱する**（一致を保証）＋**システムフォントも選択可**（選んだ場合は OS 間で差異が出る旨を UI で明示） |
| 同梱フォント構成 | **A. Noto 統一**（Noto Sans JP / Noto Serif JP / Noto Sans Mono CJK JP）。すべて SIL OFL・無償。約 6〜7MB（Regular/Bold、woff2） |
| 編集方法 | **GUI 設定パネル ＋ カスタム CSS**（上級者向けエスケープハッチ）の2層 |
| 適用範囲 | **グローバル**（アクティブな1プロファイルを全ドキュメントへ適用） |
| Light / Dark | 独立機能ではなく、プロファイル内の**配色のプリセット**として扱う |
| 標準プリセット | **ライト / ダーク / GitHub 風 / セピア**（複製・編集できる雛形。後から追加可） |
| コード/mermaid テーマ | **既定はプロファイルの colorScheme(light/dark) に連動 ＋ 個別選択可**（上級設定で固定指定） |
| 印刷 / PDF 出力 | **既定は印刷向けに変換**（紙=白背景・濃色文字。組版はプロファイル継承）。オプション「**表示通りに印刷**」で画面通り（暗背景＋明色文字）も可（§5.2） |

## 1. 基本方針：スタイルプロファイル ＋ CSS 変数

スタイルを1つの構造化データ（JSON＝**スタイルプロファイル**）として保持し、適用時に **CSS 変数（カスタムプロパティ）**へ展開する。プレビュー用スタイルシートはこの変数群のみを参照する。

```text
スタイルプロファイル(JSON)
   → CSS 変数群 (--md-*) を生成・ルートへ適用
   → プレビュー用スタイルシートが var(--md-*) を参照
   → 描画
```

- 同じプロファイル → 同じ変数 → 同じ生成 CSS となるため、**環境を問わず同一の見た目**を再現できる。
- プロファイルは**保存・複製・エクスポート/インポート**が可能（可搬性）。

## 2. 調整可能なパラメータ（カタログ）

各項目は CSS 変数に対応する。

| カテゴリ | 項目 | 対応 CSS 変数（例） |
|----------|------|----------------------|
| **全体（本文）** | フォントファミリ / 基準サイズ / 行間 / 文字色 / 背景色 / 段落間隔 / 本文最大幅 / 文字間隔 | `--md-font`, `--md-font-size`, `--md-line-height`, `--md-color`, `--md-bg`, `--md-para-gap`, `--md-max-width`, `--md-letter-spacing` |
| **見出し h1〜h6（レベル毎）** | フォント / サイズ / 太さ / 色 / 上下マージン / 下線・境界線 | `--md-h{n}-font`, `--md-h{n}-size`, `--md-h{n}-weight`, `--md-h{n}-color`, `--md-h{n}-mt`, `--md-h{n}-mb`, `--md-h{n}-border` |
| **インラインコード** | 等幅フォント / サイズ / 文字色 / 背景 / パディング / 角丸 | `--md-code-font`, `--md-code-size`, `--md-code-color`, `--md-code-bg`, `--md-code-pad`, `--md-code-radius` |
| **コードブロック** | 等幅フォント / サイズ / 背景 / 枠線 / 行間 / シンタックステーマ / 行番号有無 | `--md-pre-font`, `--md-pre-size`, `--md-pre-bg`, `--md-pre-border`, `--md-pre-line-height`, (`syntaxTheme`, `showLineNumbers`) |
| **引用(blockquote)** | 左ボーダー(色・太さ) / 背景 / 文字色 / イタリック / パディング・マージン | `--md-quote-border`, `--md-quote-border-w`, `--md-quote-bg`, `--md-quote-color`, `--md-quote-italic`, `--md-quote-pad` |
| **箇条書き（リスト）** | インデント幅 / マーカー種別 / **マーカーのサイズ・色・位置** / 番号書式 / 項目間隔 / ネスト時マーカー | `--md-list-indent`, `--md-marker-type`, `--md-marker-size`, `--md-marker-color`, `--md-marker-offset`, `--md-li-gap` |
| **リンク** | 色 / 下線 / ホバー | `--md-link-color`, `--md-link-underline`, `--md-link-hover` |
| **表(テーブル)** | 枠線 / ヘッダ背景 / ストライプ / パディング / 文字揃え | `--md-table-border`, `--md-th-bg`, `--md-table-stripe`, `--md-td-pad`, `--md-table-align` |
| **水平線(hr)** | 色 / 太さ / 余白 | `--md-hr-color`, `--md-hr-thickness`, `--md-hr-margin` |
| **画像** | 最大幅 / 枠線 / 角丸 | `--md-img-max-width`, `--md-img-border`, `--md-img-radius` |
| **チェックリスト** | チェックボックスの体裁 | `--md-checkbox-style` |
| **mermaid** | テーマ / 背景 | (`mermaidTheme`, `--md-mermaid-bg`) |

> 行番号有無・シンタックステーマ・mermaid テーマ等、CSS 変数だけで表せないものはプロファイルの設定値としてレンダリング処理に渡す。

## 3. クロスプラットフォームで同一表示にする仕組み

### 3.1 ズレの原因

1. **フォント**: OS により搭載フォントが異なり、字形・字幅が変わる。
2. **WebView エンジン差**: 本アプリは OS 標準 WebView を使うため、**Windows=Blink(WebView2) / macOS・Linux=WebKit**。UA 既定スタイルや既定マージンに微差が出る。

### 3.2 対策

| 対策 | 内容 | 効果 |
|------|------|------|
| **フォント同梱** | 既定フォントをアプリに埋め込み、`@font-face` で固定 | OS 非依存で字形・字幅が一致 |
| **CSS リセット/ノーマライズ** | UA 既定を打ち消し、全プロパティを明示指定（既定値に依存しない） | エンジン差による既定マージン等を排除 |
| **単位・ボックスモデル統一** | `box-sizing` 統一、`px`/`rem` を一貫、基準サイズ固定 | レイアウト・サイズが一致 |
| **構造化プロファイル** | 同一プロファイル→同一 CSS 変数→同一生成 CSS | どの OS でも同一の見た目を再現 |

### 3.3 同梱フォント（確定：A. Noto 統一）

本アプリは日本語 UI／日本語文書が主のため、**日本語対応（CJK）フォントを同梱**する。Noto ファミリで統一する。

| 用途 | フォント | ライセンス |
|------|----------|-----------|
| 本文ゴシック | Noto Sans JP | SIL OFL 1.1 |
| 本文明朝 | Noto Serif JP | SIL OFL 1.1 |
| 等幅（コード。**日本語も等幅対応**） | Noto Sans Mono CJK JP | SIL OFL 1.1 |

- すべて **SIL OFL 1.1・無償**。アプリへの埋め込み・再配布・商用同梱が可能（条件: フォント単体での有償販売不可／OFL ライセンス文の同梱／予約フォント名を改変版に使わない）。
- 同梱は **Regular / Bold に絞る**（Thin〜Black 全部入れると数倍に膨張するため）。
- 概算サイズ: woff2 で **約 6〜7MB**。Wails 本体が軽量なため許容範囲。
- ライセンス文（OFL.txt）をアプリに同梱する。

### 3.4 残る差と注意

レイアウト・サイズ・字形は実質同一にできるが、**アンチエイリアス／ヒンティング等の OS レベルの描画差**はごく僅かに残る（文書用途ではほぼ知覚不可）。完全なピクセル一致は保証対象外とする。

### 3.5 システムフォント選択時

ユーザーがシステムフォントを選んだ場合は一致保証の対象外となるため、**UI 上で「OS 間で表示が異なる可能性」を明示**する。

## 4. 編集方法（GUI ＋ カスタム CSS）

- **GUI 設定パネル**: カタログの各パラメータをスライダー・数値入力・色ピッカー・フォント選択で調整。変更は**ライブプレビュー**で即時反映。
- **カスタム CSS（上級者向け）**: GUI で届かない細部を直接上書き。プロファイル内 `customCSS` として保持し、生成 CSS の後段に適用する。

## 5. 適用範囲（グローバル）

- アクティブなプロファイルは1つ。**全タブ・全ドキュメントに一律適用**。
- プロファイルの切替で、開いている全ドキュメントの表示が即座に変わる。
- （将来拡張）ドキュメント毎の個別指定は必要に応じて検討。

### 5.1 コード/mermaid テーマの連動（確定）

閲覧モードの**コードブロックの構文色（highlight.js）と mermaid 図の配色**は、既定でプロファイルの `colorScheme`（light/dark）に連動させる。上級設定でプロファイル毎に特定テーマを固定指定もできる。

| プロファイル | colorScheme | 連動時のコード(highlight.js) | 連動時の mermaid |
|--------------|:---:|------------------------------|------------------|
| ライト（標準） | light | 明テーマ（例: `github`） | `default` |
| GitHub 風 | light | `github` | `default` |
| セピア | light | 明テーマ（ライト基準として扱う） | `default` / `neutral` |
| ダーク（標準） | dark | 暗テーマ（例: `github-dark`） | `dark` |

- **連動** = コード/mermaid を `colorScheme` に対応する明/暗テーマへ自動追従。
- **個別選択** = 自動連動を上書きし、プロファイル毎に固定テーマを指定（例: 本文はライトだがコードのみ暗テーマ）。
- 選択肢の例: highlight.js（`github` / `github-dark` / `atom-one-light` / `atom-one-dark` 等）、mermaid（`default` / `neutral` / `dark` / `forest` 等）。
- **注記**: ここでの対象は**閲覧モード（レンダリング表示）**のコード/図。**ソースモード（CodeMirror エディタ）の外観は別論点**（後日検討。下記 §8 参照）。

### 5.2 印刷／PDF 出力時のスタイル（確定）

印刷・PDF 出力時のスタイルは、画面の配色（特に Dark）をそのまま紙に出すと不適切なため、**既定で印刷向けに変換**する。

| モード | 背景 | 文字色 | 組版（フォント/サイズ/余白） |
|--------|------|--------|------------------------------|
| **既定（印刷向けに変換）** | 紙の色（白） | 濃色（黒／印刷用フォントカラー） | アクティブなプロファイルを**継承** |
| **「表示通りに印刷」ON** | 画面と同じ（Dark なら暗背景） | 画面と同じ（明色等） | アクティブなプロファイルを継承 |

- **既定の挙動**: 組版（フォント・サイズ・行間・余白・レイアウト）は**アクティブなプロファイルから引き継ぎ、配色だけを印刷用（白背景＋濃色文字）に差し替える**。Dark 表示中でも印刷物は読みやすい紙向けになり、白文字が白背景に乗って消える事故も防ぐ。
- **「表示通りに印刷」チェック**: 印刷ダイアログ／PDF 出力時のオプションとして提供。ON のときは画面の配色（背景色含む）を忠実に再現する。
- 実装は**印刷用 CSS（`@media print`）**で配色変数を上書きする方式を基本とする。Light 系プロファイルでは既定でもほぼ見た目が変わらない。

## 6. プロファイルの保存・共有（可搬性）

- プロファイルは OS 設定ディレクトリに JSON で保存。
- **エクスポート/インポート**に対応。別マシン・別 OS へ持ち込めば、**同梱フォントと相まって同一表示**を再現。
- 標準プリセット **ライト / ダーク / GitHub 風 / セピア** を同梱し、複製して編集できる（後から追加可）。

## 9. ソースモードのエディタ外観（確定）

ソースモード（CodeMirror 6）の外観は、「**閲覧重視・メモ帳的編集**」の方針に沿い**最小限**とする。

- **閲覧用スタイルプロファイルとは別の「アプリ共通設定」**として保持する（プロファイルへは含めない）。
- プロファイルとの連動は **Light/Dark の追従のみ**。

| 項目 | 決定 |
|------|------|
| 構文ハイライト | **控えめな Markdown 構文ハイライト**（見出し・強調・コード・リンク等を控えめに色分け） |
| 配色テーマ | **本文プロファイルの Light/Dark に連動** |
| 書体 | **等幅固定（Noto Sans Mono CJK JP）**。**サイズのみ**調整可 |
| 既定で有効な表示補助 | **行番号** / **行の折り返し（ソフトラップ）** |
| 既定で無効（設定で有効化可） | 現在行のハイライト / 空白文字の可視化 |

- すべての表示補助は設定で切替可能。
- 閲覧モードのような細かなパラメータ調整は**行わない**（最小限に留める）。
- 標準的な編集操作（元に戻す／やり直し、検索 Ctrl+F、コピー＆ペースト等）に対応。
