# Markmiru スタイル設定設計（閲覧モード）

最終更新: 2026-06-09

閲覧モードの表示スタイルを細かく調整し、かつ**クロスプラットフォームで同一表示**を実現するための設計。全体方針は [`アーキテクチャ・画面設計.md`](アーキテクチャ・画面設計.md) を参照。

## 0. 確定方針

| 項目 | 決定 |
|------|------|
| 表現方式 | **構造化「スタイル」(JSON) ＋ CSS 変数**で適用 |
| フォント | **同梱する**（一致を保証）＋**システムフォントも選択可**（選んだ場合は OS 間で差異が出る旨を UI で明示） |
| 同梱フォント構成 | **A. Noto 統一**（Noto Sans JP / Noto Serif JP / Noto Sans Mono CJK JP）。すべて SIL OFL・無償。約 6〜7MB（Regular/Bold、woff2） |
| 編集方法 | **GUI 設定パネル ＋ カスタム CSS**（上級者向けエスケープハッチ）の2層 |
| 適用範囲 | **グローバル**（アクティブな1スタイルを全ドキュメントへ適用） |
| Light / Dark | 独立機能ではなく、スタイル内の**配色のプリセット**として扱う |
| 標準プリセット | **ライト / ダーク / GitHub 風 / セピア**（複製・編集できる雛形。後から追加可） |
| コード/mermaid テーマ | **スタイルの colorScheme(light/dark) に連動**（個別固定指定は未実装・将来拡張） |
| 印刷 / PDF 出力 | **既定は印刷向けに変換**（紙=白背景・濃色文字。組版はスタイル継承）。オプション「**表示通りに印刷**」で画面通り（暗背景＋明色文字）も可（§5.2） |

## 1. 基本方針：スタイル ＋ CSS 変数

スタイルを1つの構造化データ（JSON＝**スタイル**）として保持し、適用時に **CSS 変数（カスタムプロパティ）**へ展開する。プレビュー用スタイルシートはこの変数群のみを参照する。

```text
スタイル(JSON)
   → CSS 変数群 (--md-*) を生成・ルートへ適用
   → プレビュー用スタイルシートが var(--md-*) を参照
   → 描画
```

- 同じスタイル → 同じ変数 → 同じ生成 CSS となるため、**環境を問わず同一の見た目**を再現できる。
- スタイルは**保存・複製**でき、ファイルベースの**エクスポート/インポート**（封筒形式 JSON）で別マシン・別 OS へ持ち運べる（§6）。

## 2. 調整可能なパラメータ（カタログ）

各項目は CSS 変数に対応し、GUI 設定パネルから調整できる（実装: `frontend/src/lib/style/styleDef.ts`、`components/SettingsPanel.svelte`）。

| カテゴリ | 項目 | 対応 CSS 変数 |
|----------|------|---------------|
| **全体（本文）** | フォント / 基準サイズ(px) / 行間 / 文字色 / 背景色 / 本文最大幅(px・全幅トグル) | `--md-font`, `--md-font-size`, `--md-line-height`, `--md-color`, `--md-bg`, `--md-max-width` |
| **見出し h1〜h6（レベル毎）** | フォント / サイズ(px) / 太さ(100〜900) / 色 / 上マージン(em) / 下マージン(em) / 下線(有無) | `--md-h{n}-font`, `--md-h{n}-size`, `--md-h{n}-weight`, `--md-h{n}-color`, `--md-h{n}-mt`, `--md-h{n}-mb`, `--md-h{n}-border` |
| **インラインコード** | 背景（フォント・サイズはコードブロックと共通） | `--md-code-font`, `--md-code-size`, `--md-code-bg` |
| **コードブロック** | 等幅フォント / 文字サイズ(px) / 背景 | `--md-code-font`, `--md-code-size`, `--md-pre-bg` |
| **引用(blockquote)** | 文字色 / 背景 / 左ボーダー色 / 左ボーダー太さ(px) / イタリック | `--md-quote-color`, `--md-quote-bg`, `--md-quote-border`, `--md-quote-border-width`, `--md-quote-style` |
| **箇条書き（リスト）** | インデント幅(px) / マーカー色 / マーカーサイズ(em) / マーカー位置(outside/inside) | `--md-list-indent`, `--md-marker-color`, `--md-marker-size`, `--md-list-position` |
| **リンク** | 色 / 下線(always/hover/none) | `--md-link-color`, `--md-link-deco`, `--md-link-deco-hover` |
| **表(テーブル)** | 境界線の色 / ヘッダ背景 / 奇数行背景 / 偶数行背景 | `--md-border`, `--md-th-bg`, `--md-row-odd-bg`, `--md-row-even-bg` |
| **水平線(hr)** | 色 / 太さ(px) | `--md-hr-color`, `--md-hr-thickness` |

- コード（インライン・ブロック）の**等幅フォントと文字サイズはスタイル共通**（`codeFontFamily` / `codeFontSize`）。背景のみインライン（`--md-code-bg`）とブロック（`--md-pre-bg`）で個別指定する。
- 表のストライプは**奇数行・偶数行の背景色**で表現する（ON/OFF トグルではない）。
- コードのシンタックステーマ（highlight.js）と mermaid テーマは CSS 変数ではなく、スタイルの `colorScheme`（light/dark）に連動してレンダリング処理側で切替える（§5.1）。
- 上記に加え、**カスタム CSS**（`customCSS`）で `.markdown-body` 配下を直接上書きできる（§4）。

## 3. クロスプラットフォームで同一表示にする仕組み

### 3.1 ズレの原因

1. **フォント**: OS により搭載フォントが異なり、字形・字幅が変わる。
2. **WebView エンジン差**: 本アプリは OS 標準 WebView を使うため、**Windows=Blink(WebView2) / macOS・Linux=WebKit**。UA 既定スタイルや既定マージンに微差が出る。

### 3.2 対策

| 対策 | 内容 | 効果 |
|------|------|------|
| **フォント同梱** | 既定フォントをアプリに埋め込み、`@font-face` で固定 | OS 非依存で字形・字幅が一致 |
| **CSS リセット/ノーマライズ** | UA 既定を打ち消し、全プロパティを明示指定（既定値に依存しない） | エンジン差による既定マージン等を排除 |
| **単位・ボックスモデル統一** | `box-sizing` 統一、`px`/`rem` を一貫、基準サイズ固定 | レイアウト・サイズが一致 |
| **構造化スタイル** | 同一スタイル→同一 CSS 変数→同一生成 CSS | どの OS でも同一の見た目を再現 |

### 3.3 同梱フォント（確定：A. Noto 統一）

本アプリは日本語 UI／日本語文書が主のため、**日本語対応（CJK）フォントを同梱**する。Noto ファミリで統一する。

| 用途 | フォント | ライセンス |
|------|----------|-----------|
| 本文ゴシック | Noto Sans JP | SIL OFL 1.1 |
| 本文明朝 | Noto Serif JP | SIL OFL 1.1 |
| 等幅（コード。**日本語も等幅対応**） | Noto Sans Mono CJK JP | SIL OFL 1.1 |

- すべて **SIL OFL 1.1・無償**。アプリへの埋め込み・再配布・商用同梱が可能（条件: フォント単体での有償販売不可／OFL ライセンス文の同梱／予約フォント名を改変版に使わない）。
- 同梱は **Regular / Bold に絞る**（Thin〜Black 全部入れると数倍に膨張するため）。
- 概算サイズ: woff2 で **約 6〜7MB**。Wails 本体が軽量なため許容範囲。
- ライセンス文（OFL.txt）をアプリに同梱する。

### 3.4 残る差と注意

レイアウト・サイズ・字形は実質同一にできるが、**アンチエイリアス／ヒンティング等の OS レベルの描画差**はごく僅かに残る（文書用途ではほぼ知覚不可）。完全なピクセル一致は保証対象外とする。

### 3.5 システムフォント選択時

ユーザーがシステムフォントを選んだ場合は一致保証の対象外となるため、**UI 上で「OS 間で表示が異なる可能性」を明示**する。

## 4. 編集方法（GUI ＋ カスタム CSS）

- **GUI 設定パネル**: カタログの各パラメータをスライダー・数値入力・色ピッカー・フォント選択で調整。変更は**ライブプレビュー**で即時反映。
- **カスタム CSS（上級者向け）**: GUI で届かない細部を直接上書き。スタイル内 `customCSS` として保持し、生成 CSS の後段に適用する。

## 5. 適用範囲（グローバル）

- アクティブなスタイルは1つ。**全タブ・全ドキュメントに一律適用**。
- スタイルの切替で、開いている全ドキュメントの表示が即座に変わる。
- （将来拡張）ドキュメント毎の個別指定は必要に応じて検討。

### 5.1 コード/mermaid テーマの連動（確定・実装済み）

閲覧モードの**コードブロックの構文色（highlight.js）と mermaid 図の配色**は、スタイルの `colorScheme`（light/dark）に連動させる（実装: `frontend/src/lib/style/highlight.ts`、`markdown/renderer.ts`）。

| colorScheme | コード(highlight.js) | mermaid |
|:---:|------------------------------|---------|
| light（ライト / GitHub 風 / セピア） | `github` | `default` |
| dark（ダーク） | `github-dark` | `dark` |

- **連動** = コード/mermaid を `colorScheme` に対応する明/暗テーマへ自動追従。現状はこの自動連動のみ。
- **個別固定指定**（本文はライトだがコードのみ暗テーマ等）は**未実装・将来拡張**。
- mermaid は `securityLevel: 'strict'` で初期化する（未知ソースでも安全側）。
- **注記**: ここでの対象は**閲覧モード（レンダリング表示）**のコード/図。**編集モード（CodeMirror エディタ）の外観は別管理**（§9 参照。`colorScheme` の Light/Dark のみ連動）。

### 5.2 印刷／PDF 出力時のスタイル（確定）

印刷・PDF 出力時のスタイルは、画面の配色（特に Dark）をそのまま紙に出すと不適切なため、**既定で印刷向けに変換**する。

| モード | 背景 | 文字色 | 組版（フォント/サイズ/余白） |
|--------|------|--------|------------------------------|
| **既定（印刷向けに変換）** | 紙の色（白） | 濃色（黒／印刷用フォントカラー） | アクティブなスタイルを**継承** |
| **「表示通りに印刷」ON** | 画面と同じ（Dark なら暗背景） | 画面と同じ（明色等） | アクティブなスタイルを継承 |

- **既定の挙動**: 組版（フォント・サイズ・行間・余白・レイアウト）は**アクティブなスタイルから引き継ぎ、配色だけを印刷用（白背景＋濃色文字）に差し替える**。Dark 表示中でも印刷物は読みやすい紙向けになり、白文字が白背景に乗って消える事故も防ぐ。
- **「表示通りに印刷」チェック**: 設定パネルの「印刷 / PDF（全体設定）」に置く**アプリ全体設定**（スタイルとは独立。実装: `uiStore.printAsDisplayed`）。ON のときは画面の配色（背景色含む）を忠実に再現する。
- 実装は**印刷用 CSS（`@media print`、`frontend/src/lib/styles/print.css`）**で配色変数を上書きする方式。Light 系スタイルでは既定でもほぼ見た目が変わらない。

## 6. スタイルの保存・共有（可搬性）

- スタイルは OS 設定ディレクトリの設定（`LoadConfig` / `SaveConfig`）に JSON で保存。
- 標準プリセット **ライト / ダーク / GitHub 風 / セピア** を同梱し、複製して編集できる（builtin は直接編集不可、複製して編集）。表示名は **「ライト (プリセット)」** のように接尾辞を付け、ユーザーが「ライト」等の名前を使えるようにする（`PRESET_NAME_SUFFIX`）。
- **スタイル名は全スタイルで一意**（プリセット含む。前後空白を無視した厳密一致で判定）。
  - **複製**: `「基底名」 のコピー`（プリセットからの複製は接尾辞を除いた基底名を使う）。既に在れば `… のコピー 2`, `3`… と自動で一意化（`uniqueName`）。
  - **改名**: 空・重複は**適用せず**、入力欄にインライン警告を表示して現在名へ戻す（`rename` が成否を返す）。
  - 内部 ID（`user-…`）はスタイルの安定ハンドルとして別に持ち、表示・重複判定には使わない。
- **エクスポート/インポート**（ファイルベースで別マシン・別 OS へ持ち込み、同梱フォントと相まって同一表示を再現）は**実装済み**。
  - **入口**: ネイティブメニュー「**ファイル → スタイル → インポート... / エクスポート...**」（`menu:style-import` / `menu:style-export`）。スタイル編集パネルからは独立させ、「アクティブを編集する」操作との混同を避ける。
  - **単位**: 1 スタイル＝1 ファイル（拡張子 `.json`）。
  - **ファイル形式**: メタ情報付きの封筒で包む。

    ```json
    { "app": "Markmiru", "kind": "style", "version": 1, "style": { /* Style */ } }
    ```

  - **エクスポート**: 専用ピッカー（`ExportPickerDialog`）で書き出す対象を選ばせる。**プリセット（builtin）はグレー表示で選択不可**（複製を促す）。選択後、Go 側 `ExportStyleDialog` で保存先を選び、`SaveFile` で書き込む。
  - **インポート**: `kind` が `style` であることを検証し、`normalizeStyle()` で欠損補完する。**重複は「名前」で判定**する（ユーザーから見た同一性は名前に表れ、ID は不可視のため）。取り込みは常に**新規 ID・`builtin:false`**。
    - **同名のユーザースタイルがある**: 確認ダイアログ（`confirmImportOverwrite`）で「**既に『〇〇』が存在します**」と既存名を示し「**上書き / 別名で追加 / キャンセル**」を尋ねる。上書きは既存の ID を保持して内容を置換、別名は名前を一意化して追加。
    - **同名のプリセットと衝突**: プリセットは上書きできないため、名前を一意化して追加（`uniqueName`）。
    - **衝突なし**: そのままの名前で追加。
    - いずれも取り込んだスタイルをアクティブ化する。不正なファイルはエラーダイアログ（`notify`）で通知し、状態は変更しない。
  - **実装**: 直列化/検証は `frontend/src/lib/style/styleDef.ts`（`serializeStyle` / `parseStyleFile`）、コマンドは `exportStyle` / `importStyle`（`frontend/src/lib/commands.ts`）、ピッカー状態は `frontend/src/lib/stylePicker.svelte.ts`、ストア操作は `style.svelte.ts`（`addImported` / `overwriteImported` / `findByName` / `uniqueName` / `nameTaken`）。

## 7. データ構造（実装）

実装は `frontend/src/lib/style/styleDef.ts` の `Style`（**フラット構造**）。見出しのみ h1〜h6 の配列を持つ。以下は標準プリセット「ライト」の値。

```jsonc
{
  "id": "light",
  "name": "ライト (プリセット)",
  "builtin": true,            // 標準プリセット（直接編集不可。複製して編集）
  "colorScheme": "light",     // コード/mermaid/エディタ テーマ連動に使用

  // 本文
  "fontFamily": "\"Noto Sans JP\", ...",
  "fontSize": 16,             // px
  "lineHeight": 1.7,
  "color": "#24292f",
  "background": "#ffffff",
  "maxWidth": 820,            // px
  "maxWidthFull": false,      // true でウィンドウ幅に追従（max-width: none）

  // 見出し（h1〜h6。index0 = h1）
  "headings": [
    { "fontFamily": "...", "fontSize": 32, "fontWeight": 600, "color": "#1f2328", "marginTop": 1.5, "marginBottom": 0.6, "border": true },
    // h2..h6（border は h1/h2 のみ true）
  ],

  // リンク
  "linkColor": "#0969da",
  "linkUnderline": "hover",   // "always" | "hover" | "none"

  // リスト
  "listIndent": 28,           // px
  "markerColor": "#24292f",
  "markerSize": 1,            // em
  "markerPosition": "outside", // "outside" | "inside"

  // 引用
  "quoteColor": "#57606a",
  "quoteBg": "#f6f8fa",
  "quoteBorder": "#d0d7de",
  "quoteBorderWidth": 4,      // px
  "quoteItalic": false,

  // コード（インライン・ブロックでフォント/サイズは共通、背景のみ個別）
  "codeFontFamily": "\"Noto Sans Mono\", ...",
  "codeBlockBg": "#f6f8fa",   // ブロック背景
  "codeFontSize": 14,         // px
  "codeBg": "#eff1f3",        // インラインコード背景

  // 水平線
  "hrColor": "#d0d7de",
  "hrThickness": 1,           // px

  // 表
  "borderColor": "#d0d7de",
  "tableHeaderBg": "#f6f8fa",
  "rowOddBg": "transparent",
  "rowEvenBg": "#f6f8fa",

  // 上級者向け（.markdown-body 配下を上書き）
  "customCSS": ""
}
```

- 復元時は `normalizeStyle()` で欠損項目を既定値で補完する（旧バージョン互換の安全策）。
- §2.1 の未実装項目（段落間隔・画像・チェックリスト等）はこの構造には**含まれない**。

## 9. 編集モードのエディタ外観（確定）

編集モード（CodeMirror 6）の外観は、「**閲覧重視・メモ帳的編集**」の方針に沿い**最小限**とする。

- **閲覧用スタイルとは別の「アプリ共通設定」**として保持する（スタイルへは含めない）。
- スタイルとの連動は **Light/Dark の追従のみ**。

| 項目 | 決定 |
|------|------|
| 構文ハイライト | **控えめな Markdown 構文ハイライト**（見出し・強調・コード・リンク等を控えめに色分け） |
| 配色テーマ | **本文スタイルの Light/Dark に連動** |
| 書体 | **等幅固定（Noto Sans Mono CJK JP）**。**サイズのみ**調整可 |
| 既定で有効な表示補助 | **行番号** / **行の折り返し（ソフトラップ）** |
