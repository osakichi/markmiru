# Markmiru 技術選定

最終更新: 2026-05-31

## 1. アプリ概要

Markdown ドキュメントの**閲覧・編集**を行うデスクトップアプリケーション。

- 綺麗にレンダリングして表示する「**閲覧モード**」と、Markdown 表記を直接編集する「**ソースモード**」を切り替えて使う
- どちらかといえば**閲覧に重点**を置く

## 2. 要件と優先度

| 区分 | 要件 |
|------|------|
| MUST | Markdown を綺麗にレンダリングして閲覧できる |
| MUST | mermaid を利用した図表もレンダリングして閲覧できる |
| MUST | プラグインなしの単体アプリとして動作する |
| MUST | 起動が素早い |
| MUST | マルチタブで複数ドキュメントを切り替えて表示できる |
| SHOULD | スタイルの修正/変更ができる |
| SHOULD | PDF 出力できる |
| SHOULD | クロスプラットフォーム対応 |
| SHOULD | Markdown の編集ができる（ソースモードでの編集） |
| SHOULD | 「閲覧モード」と「ソースモード」を切り替えられる |

## 3. 対象プラットフォーム

- **初回対象: Windows**
- **最終目標: Windows / macOS / Linux（デスクトップ3種）**
- モバイル（iPhone / Android）は**対象外**
  - ※ Wails 採用に伴いモバイルは不可となるため、この前提を受け入れて確定した
- 配布形態: 各プラットフォームに適した形式（例: Windows はインストーラ形式）

## 4. 技術選定の結論

| レイヤー | 採用技術 | 備考 |
|----------|----------|------|
| **実装基盤** | **Wails v2**（安定版） | Go バックエンド ＋ OS 標準 WebView |
| バックエンド言語 | **Go** | ファイル I/O、タブ/ウィンドウ管理、ネイティブメニュー、PDF 出力起動など（薄い層） |
| WebView | OS 標準 | Win=WebView2 / macOS=WKWebView / Linux=WebKitGTK |
| **フロントUI** | **Svelte + Vite + TypeScript** | コンパイル方式で軽量・高速。閲覧重視のビュワーに好適 |
| Markdown 解析 | **markdown-it** | プラグインによる拡張が豊富 |
| 図表 | **mermaid.js** | WebView 内でそのまま描画 |
| コードハイライト | highlight.js（または Shiki） | 決定は実装時に確定 |
| ソースモード編集 | **CodeMirror 6** | 行番号＋簡易ハイライト。軽量 |
| PDF 出力 | WebView の印刷 → PDF | 専用ライブラリ不要 |
| スタイル変更 | CSS（テーマ切替） | |
| マルチタブ | フロントエンド側で実装 | 1ウィンドウ内のタブバーで管理 |

## 5. 選定理由

### なぜ Wails か

- **mermaid（MUST）**: WebView 上で mermaid.js をそのまま動かせるため、最も容易に実現できる。
- **高速起動（MUST）**: Electron と異なり OS 標準 WebView を使うネイティブバイナリで、起動が速く配布サイズも小さい。
- **クロスプラットフォーム（SHOULD）**: Win/mac/Linux に対応。
- **Go で書ける**: 開発者の関心・好みに合致。
- **安定版で着手できる**: Wails は v2 が安定版。v3 は 2026 年時点でアルファのため、本プロジェクトでは **v2** を採用。

### 不採用とした主な候補

| 候補 | 不採用理由 |
|------|-----------|
| Electron | 「起動が素早い（MUST）」と衝突（重い・大きい） |
| Tauri v2 (Rust) | Wails と同等の有力候補だったが、言語の好み（Go）を優先。モバイル対応の道は閉じるが対象外のため許容 |
| Fyne / Gio (純 Go) | WebView を持たず mermaid 描画が極めて困難 |
| Flutter / .NET MAUI | mermaid 実装の難度・Linux 対応などで不利 |

## 6. 既存アプリ調査（参考）

「軽量な Markdown ビュワー」観点での既存アプリ調査の結論:

- 機能の足し算では **Typora**（有償・クローズド・WYSIWYG 主体）がほぼ全要件を満たす。
- ただし「**軽量ネイティブ ＋ マルチタブ ＋ 閲覧/ソースの明確な2モード切替**」を兼ね備えたアプリは既存に乏しく、ここが本アプリの差別化ポイント。
- 参考実装: MerMark Editor（Tauri 製）, mdr（Rust 製ビュワー）, VNote（Qt 製）。
